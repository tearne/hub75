# hub75 crate README

## Intent

The `hub75` crate has no README. Anyone landing in `hub75/` (browsing the repo, opening it on docs.rs in future, or reading on GitHub) sees the source tree with no orientation. A short README should say what the crate is for, what hardware it targets, and how to run the smoke test.

## Approach

### Orientation, not API reference

README is for someone *landing* on the crate, not someone using it day-to-day. One-paragraph summary, hardware target, quickstart for the example, and a pointer to `learning-examples/spwm_3_autonomous` as the reference architecture. API docs belong on items in `lib.rs` (when the crate goes to docs.rs), not in the README.

### Include a minimal construction snippet

A code block showing the `Dp3364sPanel::new(...)` shape lets a reader judge the API surface at a glance — what they'd need to wire up to use it.
